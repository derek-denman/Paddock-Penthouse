# F1 Provider Notes

MVP supports a flag-gated F1 provider with deterministic SIM fallback.

## Runtime Controls
- `RACE_DATA_MODE=LIVE_WITH_SIM_FALLBACK`
- `ENABLE_F1_LIVE=true`
- `F1_API_BASE=https://api.openf1.org/v1`
- `F1_SESSION_KEY=<openf1-session-key>`

If health checks fail or no session key is configured, the worker automatically falls back to SIM mode.

## Mapping
The provider emits normalized events as:
- `LAP_COMPLETED` with payload `{ lap, source: "F1_OPEN_API" }`

## Notes
- Live feeds remain provider-dependent; SIM is still the authoritative demo path.
- Historical and polling behavior are implemented for MVP reliability rather than full telemetry coverage.
